# 2 C28X处理器

## 2.1 介绍

​	C28x CPU 是32位定点处理器。该设备是具备RISC、微控制器架、固件和工具套件的最优性能的数字处理器。

## 2.3 功能

​	CPU的特点包括改进的哈佛体系结构和循环寻址。RISC的特点是单周期指令执行、寄存器对寄存器操作和修改过的哈佛体系结构。微控制器的特点包括易于使用通过一个直观的指令集，字节包装和解包，和位操作。修改后的哈佛CPU架构使指令和数据读取可以并行执行。CPU可以同时读取指令和数据，同时写入数据，以维护跨流水线的单周期指令操作。

## 2.4 浮点单元

​	C28x+浮点（C28x+FPU）处理器通过增加寄存器和指令来支持IEEE单精度浮点运算，扩展了C28x定点CPU的功能。

# 3 系统控制与中断

​	该微控制器的系统级功能，配置CPU和外设的时钟、复位和中断，以及片上存储器、定时器和安全功能的操作。

## 3.1 介绍

​	系统级配置由一组子模块控制，这些子模块统称为系统控制模块。系统控制模块提供以下功能：

* 系统级复位，包括上电和断电复位
* 时钟源选择和锁相环配置
* 时钟缺失检测
* 时钟门控低功耗模式
* 外设中断处理
* 针对某些故障条件的不可屏蔽中断
* 三个32位定时器
* 窗口看门狗定时器，它可以产生中断或复位
* RAM初始化、写保护和控制
* Flash ECC、等待状态、cache配置
* 双区代码安全模块

​	系统中的一些寄存器通过EALLOW保护机制来防止破坏性的CPU写入。它使用特殊的CPU指令EALLOW和EDIS来启用和禁用对受保护寄存器的访问。当前的保护状态由CPU ST1寄存器中的EALLOW位给出，如下表3-1所示：

## 3.5 外设中断

​	本节解释设备上的外设中断处理。3.6节中包含了不可屏蔽的中断。软件中断和仿真中断不包括在本文档中。

### 3.5.1 中断概念

​	中断是一个信号，它导致CPU暂停当前的执行，并分支到称为中断服务例程（ISR）的另一段代码。这是处理外设事件的有用机制，并且比寄存器轮询涉及更少的CPU开销或程序复杂性。但是，由于中断对程序流是异步的，因此必须小心避免在中断和主程序代码中访问的资源上发生冲突。

​	中断通过一系列标志寄存器和使能寄存器传播到CPU。标志寄存器存储中断，直到它被处理。使能寄存器阻止中断的传播。当中断信号到达CPU时，CPU从一个称为**向量表**的列表中获取适当的ISR地址。

### 3.5.2 中断架构

​	C28x CPU有14条外设中断线。其中两个（INT13和INT14）分别直接连接到CPU定时器1和2。其余12个通过增强型外设中断扩展模块（ePIE，或简称PIE）连接到外设中断信号。PIE将多达16个外设中断多路复用到每条CPU中断线上，并扩展向量表以允许每个中断拥有自己的ISR。这使得CPU可以支持大量的外设。

​	中断路径分为三个阶段——外设、PIE和CPU。每个阶段都有自己的使能和标志寄存器。该系统允许CPU处理一个中断，而其他正在等待，在软件中实现和优先级嵌套中断，并在某些关键任务期间禁用中断。

#### 3.5.2.1 外设阶段

​	每个外设都有自己独特的中断配置，这在该外设的章节中进行了描述。有些外设允许多个事件触发相同的中断信号。例如，通信外设可能使用相同的中断来指示数据已被接收或出现传输错误。中断的原因可以通过读取外设的状态寄存器来确定。通常，在产生另一个中断之前，必须手动清除状态寄存器中的位。

#### 3.5.2.2 PIE阶段

​	PIE为每个外设中断信号提供单独的标志和使能寄存器位，这些位有时被称为PIE通道。这些通道根据它们相关的CPU中断进行分组。每一个PIE组有一个16位的使能寄存器（PIEIERx），一个16位的标志寄存器（PIEIFRx）和一个在PIE确认寄存器（PIEACK）中的位。PIEACK寄存器位作为整个PIE组的公共中断掩码。

​	当CPU接收到中断时，CPU从PIE中获取ISR的地址。PIE返回标志且使能的组中编号最低的通道的向量。当多个中断挂起时，这给编号较低的中断一个较高的优先级。

​	如果没有标志和使能中断，则PIE返回通道1的向量。除非中断传播时软件改变了PIE的状态，否则这种情况不会发生。第[3.5.4节] (###3.5.4 配置与使用中断) 包含在启用中断后安全修改PIE配置的过程。

#### 3.5.2.3 CPU阶段

​	与PIE一样，CPU为每个中断提供标志和启用寄存器位。有一个启用寄存器（IER）和一个标志寄存器（IFR），它们都是CPU内部寄存器。还有一个全局中断掩码，它由ST1寄存器中的**INTM**位控制。这个掩码可以使用CPU的**SETC**和**CLRC**指令来设置和清除。在C代码中，C2000Ware的**DINT**和**EINT**宏可用于此目的。

​	对IER和INTM的写入是原子操作。特别是，如果设置了INTM，流水线中的下一条指令将在禁用中断的情况下运行。不需要软件延迟。



### 3.5.3 中断进入序列

外设中断传播到CPU的过程如图3-2所示。

![image-20251205164338070](C:\Users\10637\AppData\Roaming\Typora\typora-user-images\image-20251205164338070.png)

​						 Figure 3-2. Interrupt Propagation Path

当外设产生中断（在PIE组x上，通道y上）时，它触发以下事件序列：

1. 中断锁存在 **PIEIFRx .y**中；
2. 如果 **PIEIERx .y**置 1，中断传递；
3. 如果 **PIEACK.x**置 0，中断传递并且 **PIEACK.x**置 1；
4. 中断锁存在 **IFR.x**；
5. 如果 **IER.x**置 1，中断传递；
6. 如果 **INTM** 置 0，CPU接收该中断；
7. **D2**或后续阶段的任何指令都将运行至完毕。较早阶段的指令被刷新；
8. CPU存储上下文到栈中；
9. **IFR.x** 和 **IER.x** 被清除，**INTM** 置 1，**EALLOW** 清除；
10. CPU 从 PIE中获取ISR的向量，**PIEIFRx.y** 被清除；
11. CPU 跳转分支到 ISR。

​	中断延迟是 **PIEIFRx .y**锁存中断和进入 CPU 流水线执行阶段的第一条 ISR 指令之间的时间。最小中断延迟为 **14 SYSCLK**周期。ISR 或堆栈内存上的等待状态将增加延迟。外部中断为 GPIO 同步添加了至少 **2 SYSCLK** 周期，输入确认添加了额外的时间（如果使用的话）。使用C28x RPT指令创建的循环不能被中断。
